3D Printing from the web (OctoPrint)

3D printing is rewarding and useful. Even more so when it can be done from anywhere in the world! Thanks to the open-source framework OctoPrint, all it takes is a Raspberry Pi and a little Linux knowledge to start and monitor 3D prints from any web browser.

If the Printrbot in my garage is currently online, tune in via the 'Control' tab to see if something is in the works!

BoomBox [pics]

I really enjoy going to the shooting range and punching some holes in some paper. My friend (@psionicpotato) and my roommate (@theSadnap) and I thought it'd be fun to design a device you could attach to a pistol to log motion data and begin extrapolating 'good' and 'bad' habits in order to help people improve and challenge their friends!

Using our 3D printer, the PJRC Teensy 3.0, the Bluegiga WT12, and Android Studio, we got a working, but very rough prototype which streamed motion data to an Android smartphone! Using Python, we began writing some signal analysis code to process the input.

This is where we stopped, but this spun off an idea which I continued to run with: a web application which automatically scores pictures of targets which you've shot, ShotLog.us.

ShotLog.us

Since I like target shooting, I made this web application for target-shooters to automatically score images of a standard pistol target.

It's currently up on the web and was a great learning experience of how to use Linux cloud servers via AWS EC2, cloud storage via AWS S3, Ruby on Rails, and OpenCV for target scoring. Though it currently only has the core feature of 'scoring' the target, at some point I'll come back and implement the remaining core features: improvement tracking and friend leader-boards!

Feel free to give it a try! Print out a few of these, put them at 21 ft at a range, and take 5 shots per target!

A special thanks to my friend @mjsteger for helping me glue the code together, and to Michael Hartl's Rails Tutorial, which serves as the foundation to the application.

Mechanical Keyboards

I like to type on and make mechanical keyboards. Two I've done "build-to-print", but the third will be a new design!

MicroMechBoard [pics]

I stumbled across an Open Source Hardware project of a small, split-hand mechanical keyboard. The project code and hardware files for @wyager's MicroMechBoard are all excellent and posted on GitHub. I followed along with his instructions and blog post and the keyboard came out exactly as advertised! This was a huge discovery for me, giving me a great starting point to inexpensive, small-scale prototype development.

ErgoDox [pics]

Sometime later, I built another split-hand mechanical keyboard called the ErgoDox for my friend (the one who helped me with ShotLog, in fact!), and it was a wonderful experience. I learned a lot about sourcing components and refined my solder-reflow techniques. As this was my second build-to-print keyboard, I decided to get a little more adventurous with the design with my next one.

mGH60 (to be the 'ble60') [pics]

Thanks to the helpful and open mechanical keyboard community, I forked @komar007's GH60 keyboard design. I've started by shrinking the hardware onto a small, reprogrammable board which I can use to begin debugging a breadboard prototype. I've named the reprogrammable board the 'mGH60', a 'miniature' version of the original.

I've shared the project on OSH Park so anyone can download or fabricate the mGH60 too! The source files are in my GitHub repository under ghkb, forked from @komar007.

The next step will begin playing around in the tmk_keyboard code to get the mGH60 communicating with the BLE113 Bluetooth 4.0 module I already have. If all goes well, a 60% PCB design will be next, the ble60!
